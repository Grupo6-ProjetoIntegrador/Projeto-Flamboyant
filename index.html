Visão Geral do Projeto

Este é um SPA React (Vite + TypeScript) seguindo arquitetura MVVM. Gerencia propostas comerciais de locação, disponibilidade de lojas e relatórios do shopping JP Mall.

Nesta página

  1. Stack tecnológica
  2. Rotas disponíveis
  3. Fluxo de dados
  4. Convenções de nomenclatura

Stack tecnológica

⚛️ React 18 + TypeScript

Componentes funcionais, hooks. Sem classes.

⚡ Vite 6

Build tool. Dev server com proxy para API em /api/v1.

🎨 Tailwind CSS 4

Utility-first. Configuração via postcss.config.mjs.

🔀 React Router 7

Roteamento. BrowserRouter + nested routes.

📊 Recharts

Gráficos de barras, pizza e linha nos dashboards.

🎯 Radix UI + shadcn

Primitivos acessíveis: Dialog, Select, Tabs, etc.

📄 jsPDF + xlsx

Exportação de relatórios em PDF e Excel.

🖼️ Lucide React

Ícones SVG. Sempre com strokeWidth={1.5}.

Convenções de nomenclatura

  • Componentes: PascalCase (DataCard, KpiContainer)
  • Hooks/ViewModels: camelCase com prefixo use (useComercialProposals)
  • Serviços: PascalCase + sufixo Service (PropostasService)
  • Enums de valor: SCREAMING_SNAKE_CASE (STATUS_APROVADO)
  • Tipos TypeScript: PascalCase (Proposta, TipoProposta)
  • Chaves sessionStorage: namespace ponto campo (propostas.filterStatuses)
🔑
Modo Protótipo

O sistema roda em modo protótipo: qualquer credencial funciona no login, o token é fixo ('proto-token') e não há autorização real. Para produção, veja as instruções em AuthContext.tsx.

Proxy em desenvolvimento

O Vite está configurado com proxy em vite.config.ts: requisições para /api/v1/* são redirecionadas para http://localhost:8080, evitando problemas de CORS.

Estrutura do vite.config.ts

import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [react(), tailwindcss()],
  server: {
    proxy: {
      '/api/v1': 'http://localhost:8080',
    },
  },
})

⚠️
204 No Content

Endpoints que retornam 204 (ex: atualizarStatus, remover) retornam undefined. Não tente desestruturar o resultado.

Dados

useApi (hook)

Hook genérico que gerencia o ciclo de vida de uma requisição: loading → data | error, com suporte a refetch e proteção contra race conditions.

Assinatura

useApi<T>(
  fetcher: () => Promise<T>,
  deps: any[] = []
): { data: T | null, loading: boolean, error: string | null, refetch: () => void }

Uso básico

import { useApi } from '../data/useApi';
import { propostas } from '../data/apiClient';

function MinhaPage() {
  const { data, loading, error, refetch } = useApi(
    () => propostas.listar(),
    []  // re-executa apenas na montagem
  );

  if (loading) return <Spinner />;
  if (error)   return <ErroMsg>{error}</ErroMsg>;

  return <Lista items={data ?? []} />;
}

Com deps — rebusca quando muda

const [filtroStatus, setFiltroStatus] = useState('');

const { data } = useApi(
  () => propostas.listar({ status: filtroStatus }),
  [filtroStatus]  // re-executa quando filtroStatus muda
);

Forçar refetch manualmente

const { data, refetch } = useApi(() => propostas.listar(), []);

// Após salvar, re-busca a lista
const handleSalvar = async () => {
  await propostas.atualizar(id, draft);
  refetch();  // ← dispara nova requisição
};
🛡️
Proteção contra race condition

O hook usa um contador interno. Se uma nova requisição for disparada antes da anterior terminar, a resposta antiga é descartada. Não é necessário nenhum tratamento extra.

Anatomia de um ViewModel

export function useComercialProposals() {
  // 1. DADOS — busca da API
  const { data, refetch } = useApi(() => PropostasService.listar(), []);

  // 2. FILTROS — persistidos em sessionStorage
  const [filterStatuses, setFilterStatuses] = usePersistedState(
    'propostas.filterStatuses', []
  );

  // 3. DADOS DERIVADOS — pipeline de filtro + sort
  const filtered = useMemo(() =>
    proposals.filter(p => matchStatus(p) && matchDate(p)),
    [proposals, filterStatuses, dateFrom, dateTo]
  );

  // 4. UI — modal aberto
  const [modalAberta, setModalAberta] = useState<Proposta | null>(null);

  // 5. RETORNO — tudo que a View precisa
  return {
    tableRows: filtered,
    filterStatuses, setFilterStatuses,
    modalAberta, setModalAberta,
    refresh: refetch,
  };
}

Criar um novo ViewModel

// src/app/viewmodels/useMinhaPagina.ts
import { useState, useMemo } from 'react';
import { useApi } from '../data/useApi';
import { minhaEntidade } from '../data/apiClient';
import { usePersistedState } from '../shared/hooks/usePersistedState';

const NS = 'minha-pagina'; // namespace para sessionStorage

export function useMinhaPagina() {
  const { data, refetch } = useApi(() => minhaEntidade.listar(), []);
  const [filtro, setFiltro] = usePersistedState(`${NS}.filtro`, '', v => v, v => v);
  
  const items = useMemo(() =>
    (data ?? []).filter(i => !filtro || i.nome.includes(filtro)),
    [data, filtro]
  );

  return { items, filtro, setFiltro, refresh: refetch };
}

Utilitários

usePersistedState

Hook que funciona como useState, mas persiste o valor no sessionStorage automaticamente. Usado para manter filtros e preferências de UI entre navegações.

Assinatura

usePersistedState<T>(
  key: string,
  defaultValue: T,
  serialize?: (v: T) => string,   // padrão: JSON.stringify
  deserialize?: (s: string) => T  // padrão: JSON.parse
): [T, Dispatch<SetStateAction<T>>]

Exemplos

// Array (usa JSON por padrão)
const [filtros, setFiltros] = usePersistedState<StatusProposta[]>(
  'propostas.filterStatuses', []
);

// String — use identity para evitar double-encoding
const [busca, setBusca] = usePersistedState<string>(
  'propostas.busca', '',
  v => v,    // serialize: string → string
  v => v     // deserialize: string → string
);

// Enum customizado
const [viewMode, setViewMode] = usePersistedState<ViewMode>(
  'propostas.viewMode', ViewMode.Cards,
  v => v,
  v => v as ViewMode
);
⚠️
Sempre use string identity para strings

Se usar JSON.stringify em uma string, o valor salvo fica com aspas extras ("\"valor\""). Passe v => v como serialize e deserialize para strings simples.

Criar Modal de Edição

Como criar um modal de manutenção completo seguindo o padrão do projeto.

import { useState } from 'react';
import { Save, X } from 'lucide-react';
import {
  ManutencaoModalShell,
  ManutencaoToolbar,
  ToolbarBtn,
  ToolbarDivider,
  TabBar,
  InfoHeaderBar,
  HeaderField,
} from '../ManutencaoShared';
import { ConfirmModal } from '../ConfirmModal';

interface Props {
  item: MinhaEntidade;
  onClose: () => void;
  onSave: (item: MinhaEntidade) => void;
}

export function MinhaEntidadeModal({ item, onClose, onSave }: Props) {
  const [draft, setDraft] = useState({ ...item });
  const [editMode, setEditMode] = useState(false);
  const [activeTab, setActiveTab] = useState('dados');
  const [confirmSave, setConfirmSave] = useState(false);

  const abas = [
    { id: 'dados', label: 'Dados' },
    { id: 'historico', label: 'Histórico' },
  ];

  return (
    <>
      <ManutencaoModalShell onClose={onClose}>
        <ManutencaoToolbar>
          {!editMode && (
            <ToolbarBtn icon={Pencil} label="Editar" onClick={() => setEditMode(true)} />
          )}
          {editMode && (
            <>
              <ToolbarBtn icon={Save} label="Salvar" variant="primary"
                onClick={() => setConfirmSave(true)} />
              <ToolbarDivider />
              <ToolbarBtn icon={X} label="Cancelar"
                onClick={() => { setDraft({ ...item }); setEditMode(false); }} />
            </>
          )}
        </ManutencaoToolbar>

        <InfoHeaderBar>
          <HeaderField label="Nome">{draft.nome}</HeaderField>
        </InfoHeaderBar>

        <TabBar tabs={abas} activeTab={activeTab} onTabChange={setActiveTab} />

        {activeTab === 'dados' && (
          <div className="p-4">
            {/* campos de edição */}
          </div>
        )}
      </ManutencaoModalShell>

      <ConfirmModal
        isOpen={confirmSave}
        title="Salvar alterações?"
        message="Confirmar as mudanças realizadas?"
        onConfirm={() => { onSave(draft); setConfirmSave(false); setEditMode(false); }}
        onCancel={() => setConfirmSave(false)}
      />
    </>
  );
}

Erros Comuns

Problemas frequentes e como resolvê-los.

❌ useAuth must be used inside AuthProvider

Você está usando useAuth() em um componente fora da árvore do AuthProvider. Certifique-se que o componente está dentro do <App>, que já envolve tudo com AuthProvider.

❌ Double-encoded string no sessionStorage

Você usou usePersistedState<string> sem passar as funções de serialização identity. Solução:

// ❌ Errado:
const [val, setVal] = usePersistedState<string>('key', '');

// ✅ Correto:
const [val, setVal] = usePersistedState<string>('key', '', v => v, v => v);

❌ CORS ao chamar a API diretamente

Em desenvolvimento, use sempre o caminho relativo /api/v1/... (o Vite proxy redireciona). Nunca chame http://localhost:8080 diretamente no componente. Configure o VITE_API_URL no .env.

❌ Tailwind dark: não funciona

O dark mode é ativado via classe dark no elemento <html> (configuração padrão do Tailwind v4). Verifique se o Layout está corretamente adicionando/removendo essa classe.

❌ DataTable não exibe coluna

A coluna é inferida das chaves do primeiro objeto do array data. Se o array estiver vazio, nenhuma coluna é criada. Certifique-se de passar data={items ?? []} e que _specified não está false para aquela coluna.

❌ Modal não fecha ao clicar no overlay

O ManutencaoModalShell fecha via prop onClose ao clicar fora. Se o modal não está fechando, verifique se você está passando onClose corretamente e se não há um e.stopPropagation() desnecessário no conteúdo interno.

❌ evento navigate-proposta não abre o modal

O listener existe no useComercialProposals. Garanta que a página de Propostas já está montada (o usuário navegou até ela pelo menos uma vez). O evento é descartado se o componente não estiver montado.

❌ Filtro não persiste após navegação

Verifique se a chave do usePersistedState é única e segue o padrão 'namespace.campo'. Chaves duplicadas causam conflito de estado entre páginas.

🔍
Depurar o sessionStorage

No Chrome DevTools → Application → Session Storage → localhost. Você pode ver e editar manualmente todos os valores persistidos pelos ViewModels.
